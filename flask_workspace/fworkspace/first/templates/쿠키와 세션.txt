쿠키와 세션 
쿠키 - 클라이언트 컴퓨터에 정보가 저장된다. 저장방식은 dict타입과 유사 
       키와 값 형태로 데이터를 text파일에 저장한다 
       예전에는 이걸 이용해서 백도어 바이러스등을 심어서 납치를 해간다
       PHP라는 언어는 초창기에 로그온 정보를 저장할 세션이 없어서
       쿠키 쓰다가 보안에 너무 취약 
       서버가 클라이언트를 공격하는데 쓰임, 지금은 최소한도로 사용하고
       있다. 

세션 - 서버에 저장된다. (로그온 정보나 장바구니 등의 정보를 저장하기 위해 쓴다 )
       각 클라이언트마다 하나의 세션이 만들어지고 이 세션은 서로 
       침범하지 않는다. 세션에 너무 많은 정보를 저장할 경우 서버의
       부담이 커진다. 가져와야할 정보가 클경우에는 디비에 저장해놓고 
       그때 그때 갖다가 쓴다. 언어별로 세션을 서버의 파일이나 메모리
       디비등에 저장해서 사용하는 경우도 있다 

캐시 -  우리가 해야할 공부가 많다. 내가 봐야할 책들을 바로 옆에 쌓아놓고
        바꿔가면서 공부 한다. 

        메모리 ----> cpu(일이 끝나면)
              <----

        RAM(DRAM):주기적으로 재충전을 안해주면 메모리가 날라간다. 
        메모리------>  캐시메모리(SRAM)  -------> cpu
